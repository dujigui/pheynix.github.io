MP实现技术总结

2016-02-29

MP是一个OA（办公自动化）APP，包括任务管理、联系人、审批、CRM等功能点，主要面对的是企业管理。整个项目所用到的东西普通实用，这里主要总结一下整个项目用到的这些Android开发的经验。所有文章在个人时间完成，所有的代码展示并非项目源代码，而是根据需要重新编写。

在这个项目中，网络访问使用Retrofit和OkHttp，图片加载选择Picasso。
数据请求回来之后，本地使用LiteOrm操作数据库，辅以SharePreferences保存键值对数据，然后将恰当的向上传递。

为了简化代码和加快开发数据，使用了ButterKnife完成View的绑定和AndroidAutoLayout解决屏幕适配的问题。

一般来说，Activity/Fragment之间传递数据使用setArgument()/setResult()或者回调接口的方法，然而我们可以使用事件总线RxBus的办法，简化整个传递过程并且可以获得更大的灵活性。

APP运行过程中经常要用到当前用户的数据，例如token、用户账号信息以及是否已登录等等，与其每次使用都到SharePreferences中读取，还不如在启动的时候加载到应用中，通常的做法是建立UserConfig单例在运行期间持有这些信息。具体的使用可以看Userconfig使用或者看DrKLO大神的表演Telegram UserConfig

很多时候我们需要在应用启动的时候做一些初始化的工作，可以选择继承Application类，在这个类中初始化项目要用到的liteOrm等等。

目前比较流行的网络请求框架有OkHttp/Retrofit、android-async-http、volley等，在这个项目选用的是第一个，可以很方便地分离主线程和网络线程，并且使用函数式编程，如果只做简单的网络请求，一些OkHttp/Retrofit的基本用法已经可以满足需求。

数据取回来之后，需要进行保存，一些简单的数据可以保存在SharePreferences中，对于很多项目，都需要一些工具类，减少重复的代码。对于更大更复杂的数据，就需要用到数据库了，可以直接上DataBaseHepler，或者偷懒用ORM。对于Orm，目前流行的可以实用GreenDAO和liteOrm，在ManagerPlus中选用的是LiteOrm。

使用LiteOrm的时候，最好能够继承Application并且在子类中创建LiteOrm单例，这样就可以在整个项目范围方便地访问数据库了。具体使用可以看：

  • LiteOrm：Android高性能数据库框架
  • 使用案例1
  • 使用案例2

在程序运行的过程中，不同的Activity/Fragment之间需要交换数据，可以使用EventBus或者RxBus.
既然我们的项目中已经有RxJava了，就可以使用一个简单的单例类实现RxBus，具体方法可以看RxBus—通过RxJava来替换EventBus

创业公司很大程度要求快速开发，难免使用流行可靠的开源项目，自然对员工的快速学习能力有很高的要求。怎么[挑选、使用开源项目](www.opensource.com]对项目今后的发展有很大程度的影响。

项目使用的IDE是Android Studio，相对于Eclipse来说，提供很多贴心的功能。每天的Tips of Today都会带来不一样的惊喜！
。这里有很多可以提高开发效率的小技巧Android Studio Tip

对于软件项目来说，版本管理必不可少，Git是一个很好的选择。
Git可以直接使用terminal命令，也可以用sourceTree等一些GUI 客户端。Git基础的一些操作可以在这里找到中文教程。

Git中的一个重要的概念就是分支，分支可以帮助我们将新特性、漏洞修复、发布和主分支区别开来，在团队合作中，遵循普遍的分支模型可以减少很多沟通成本。合并的时候如果出现冲突，建议使用Android Studio的solve conflict功能（osx的快捷键ctrl+c）。

在调试API接口的时候，我们可以使用Postman这款Chrome插件，如果是osx还能使用客户端，非常方便。

微信SDK使用过程问题

2016-02-22

Wechat SDK 提供了授权登录/分享/支付等功能，作为国内用户量很大的一款即时通讯APP，其他的App接入Wechat的相关功能是一个很正常的需求，接入过程网上可以找到不错的教程，其中的几个坑如果能够避开可以节省不少时间。

坑1 无法接收授权登录结果

按照官方的教程，无论是授权登录还是分享功能的操作结果都是通过WXEntryActivity.java这个在wxapi包下的Activity返回。

在教程中有一句话：

在WXEntryActivity中将接收到的intent及实现了IWXAPIEventHandler接口的对象传递给IWXAPI接口的handleIntent方法

这句话中的intent不仅仅指代onNewIntent(Intent intent)回调中的intent，更重要的是，也包括初次创建此Activity时通过在onCreate()回调通过getIntent()方法拿到的intent。
如果发现用户确认授权之后收不到结果，检查是否在onCreate()中将intent传递给IWXAPIEventHandler了。

坑2 多包名适配

前面说到 Wechat SDK 返回的结果依赖于包名和类名，当业务需要根据不同的包名来区分不同的Channel时（例如me.pheynix.test.dev和me.pheynix.test区分开发版和稳定版），由于包名的改变应用签名改变自然不需要多说。真正的问题在于，包名的改变导致操作结果再次无法接收到。
既然返回的结果依赖于包名和类名，那么修改WXEntryActivity.java所在包即可解决问题，最后包的结构类似：


me.pheynix.test
├── dev
│   └── wxapi
│       └── WXEntryActivity
└── wxapi
    └── WXEntryActivity

虽然可以解决问题， 这种方法更像一种Workaround而不是Solution。

怎么选择使用开源项目

2016-02-22

很多时候，限于项目进度要求和避免重复造轮子的浪费，或多或少会使用到开源项目，怎么快速挑选、使用恰当的开源项目很大程度影响项目日后的开发.

收集、选择现有能满足需求的项目框架

例如网络访问框架，使用“android网络请求框架”和“android network”等关键词进行搜索之后就可以大致得到当下流行的一些网络请求框架：

  • Retrofit
  • OkHTTP
  • Volley
  • android-async-http
  • Afinal
  • xUtils
    一般考虑易用性、符合项目需求程度、用户数量、项目更新频次等因素。原则上满足需求前提下，尽量选择符合标准（例如HTTP标准）、功能专注且作者还在积极维护的项目。

查阅文档和教程，熟悉使用方法

一个优秀的开源项目，必定会提供有相关的项目文档和sample项目。
以Retrofit为例，项目的github仓库中的README.md中有明确的项目定位说明、导入方法和使用限制

Retrofit requires at minimum Java 7 or Android 2.3.

项目中也附带了sample项目，可以直接导入到Android studio运行和阅读。
在项目的主页中，作者对该项目的调用方法做了详尽的说明。基本上，阅读完这些材料之后已经可以上手这个开源框架了。如果实在还没有头绪，可以使用中文搜索引擎搜索中文的相关使用博客文章。

引入、使用项目

使用Android Studio之后，引入一个开源框架的方法大致有三种：

  • 添加Maven/Gradle依赖
  • 下载导入JAR文件
  • 导入项目源文件作为module

对于添加依赖的方法，以Gradle为例，在原项目中的build.gradle中的dependencies中添加依赖语句即可。

对于JAR文件，下载回来之后，复制到/libs文件夹中，然后在Android Studio中右键点击并选择Add as library，到gradle.build如果发现有一句compile files('libs/lite-orm-1.7.0.jar')说明导入成功。

对于第三中导入项目源文件作为module的方法，稍微复杂一些，但是带来的优点是可以按照项目需求修改导入的开源框架的代码，拥有更大的主动权。
以HeadsUp项目为例，使用git clone https://github.com/zzz40500/HeadsUp.git将项目克隆到本地之后，依次选择File -> New -> Import module，然后选择想要导入的library即可。
然后根据项目需求，做一些修改即可。

定义工具类减少项目的重复代码

2016-01-28

根据三振法,

  • The first time you do something, you just do it.
  • The second time you do something similar, you wince at the duplication, but you do the duplicate thing anyway.

  • The third time you do something similar, you refactor.

  • 某件事第一次做的时候不需要考虑

  • 第二次做的时候，或许不喜欢复制的方式，不过也没有太大的问题
  • 第三次做的时候，就需要考虑重构了

如果发现某段代码被复制了2次，那么就可以重构成工具类，方便日后的使用。
在开发Android应用的过程中，普遍需要编写一些工具类例如：与文件有关的FileUtil 、与日期相关的DateUtils；Android系统提供的TextUtil和AnimationUtils等
经常需要使用的SharePreferences用于保存键值对，利用工具类可以减少获取Editor/SharedPreferences对象的重复代码。

Java注释基础用法

2016-01-21

Java支持三种类型的注释， 两种会被JavaDoc忽略掉，分别是/* some comment here */和// some comment here；第三种注释叫doc comment，用于生成文档：

常用的说明注释标签

标签 描述
@author authorName 类的作者
@deprecated 已弃用类
@param paramName description 参数
@return valueName description 返回值
@exception exceptionName description 抛出的异常
@throws exceptionName description 抛出的异常
{@link className} 链接到另一个类
@see className 链接到另一个类
@since date 发布日期
{@inheritDoc} 继承父类注释
