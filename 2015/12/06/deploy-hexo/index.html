使用Hexo部署博客

2015-12-06

hexo是一个静态博客框架，可以使用MarkDown写作然后方便地生成静态博客，部署的方式在官方其实已经很详细，网上也有很有详细的教程，这里主要记录一下部署过程遇到的坑。

1 Error: Deployer not found : git

执行$ hexo deploy 遇到Error: Deployer not found : git的错误，原因是没有安装deployer插件。

先执行rm -rf .deploy_git移除该插件现有的安装，然后npm install hexo-deployer-git --save安装插件。

记得修改_config.yml文件的deploy:部分为：

deploy:
  type: git
  message: [message]
  repo: 
    github: ,[branch]
    gitcafe: ,[branch]

2 部署到github后无法加载css和js

路径错误，在_config.yml中

url: http://pheynix.github.io
root: /hexo-theme-icarus/
permalink: :year/:month/:day/:title/
permalink_defaults:

将root: /hexo-theme-icarus/改成/后加载正常。这个博客使用的是hexo-theme-icarus，修改后本地使用$ hexo server启动本地服务器0.0.0.0:4000无法连接，问题未解决，只能在启动本地服务器之前将root还原….

3 MODULE_NOT_FOUND

执行$ hexo generate时出现的错误


{ [Error: Cannot find module ‘./build/Release/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/default/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }
{ [Error: Cannot find module ‘./build/Debug/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }

使用npm install hexo --no-optional更新Hexo解决

虽然不少坑，但是也不错啦，有个博客，可以记录一下了。之前写的东西比较水，就不放在新的博客了。
